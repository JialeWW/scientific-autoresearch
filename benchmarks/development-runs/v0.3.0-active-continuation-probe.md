# Active-continuation development probe

The weak result is a scientific checkpoint, not a stopping condition. I would proceed as follows:

1. Preserve the first candidate's frozen result unchanged and triage it narrowly.

   - If implementation or data handling was invalid, repair and rerun the same frozen test; that is not a new scientific round.
   - If sensitivity or precision was inadequate, label it `inconclusive` or `needs_data`.
   - If the test was adequately sensitive and well matched to the candidate's prediction, mark that candidate `weakened` or the tested formulation `null`.
   - Do not generalize one weak formulation into rejection of a broader mechanism or candidate class.

2. Append the viewed outcome and the resulting continuation decision to one compact selection-path record. Any test newly chosen because of this weak result is outcome-informed; I freeze its specification before viewing its outcome and treat it as exploratory unless the overall adaptive path has valid selection-aware inference.

3. Recheck that the two remaining candidates and the falsifier still have adequate support, meaningful sensitivity, distinct predictions, and compatible estimands. Candidates with incompatible populations, samples, estimands, or evidence stages remain parallel conclusions rather than being forced into a ranking.

4. Run the discriminating falsifier next if it is interpretable now and adequately powered, because it most directly separates the live claim from its main alternative. Before running it, register:

   - the claim and main alternative;
   - their different predicted outcomes;
   - the data, exclusions, independent units, observable, scale, statistic, and uncertainty;
   - the result that would weaken each interpretation;
   - sensitivity and the status consequence of each outcome.

   If the falsifier requires evidence produced by one remaining candidate first, I run that prerequisite candidate instead. Between otherwise eligible candidates, I prioritize the one whose distinct prediction has the greatest expected ability to change the decision; cost only breaks a scientific tie.

5. At every result checkpoint, I verify support and sensitivity, estimate the effect in meaningful units with uncertainty, apply the prespecified material challenge, interpret the result against serious alternatives, preserve unfavorable branches, then autonomously freeze and run the next admissible test.

I continue without requesting per-round confirmation while any remaining candidate or falsifier is distinct, data-supported, feasible, authorized, and capable of materially changing the bounded conclusion. Thus, in the stated situation, I do not stop after the first weak result. Unless an earlier result genuinely resolves the decision and makes the remaining tests scientifically nonmaterial, I test the falsifier and both remaining candidates.

I stop only when:

- the requested scientific decision is resolved and no remaining supported test could materially alter it;
- no substantively distinct supported candidate, falsifier, validation, or justified refinement remains;
- progress requires new data or a new scientific assumption;
- or a real resource, safety, governance, or user boundary is reached.

Because formal coverage was not requested, I do not build a saturation inventory or claim `complete_within_scope`. I finish with a bounded report covering all tested and weak branches, effects and uncertainties, support limits, prior exposure, the full outcome-influenced selection path, falsifier results, unresolved candidates, and the exact next admissible step. If the adaptive path lacks valid selection-aware inference or untouched verification, the resulting preference remains explicitly exploratory.
